## 📌 Project Overview

This project simulates a **real-world food delivery platform** (inspired by Zomato) using a fully normalized relational database built from scratch in MySQL. The goal is to perform **end-to-end SQL-based business analysis** — covering customer behavior, restaurant performance, rider operations, and revenue trends — using 18 analytical queries.

---

## 🗄️ Database Schema

The database `zomato_db` consists of **5 interrelated tables**:

| Table | Description |
|---|---|
| `customers` | Customer profiles with registration date |
| `restaurants` | Restaurant info with city and opening hours |
| `orders` | Core transactional table — items, status, amount |
| `riders` | Delivery rider details |
| `deliveries` | Delivery status and time per order |

### 🔗 Entity Relationship Diagram

```
customers ──< orders >── restaurants
                │
            deliveries >── riders
```

> **Foreign Keys:** `orders.customer_id → customers`, `orders.restaurant_id → restaurants`, `deliveries.order_id → orders`, `deliveries.rider_id → riders`

## 🧹 Data Cleaning

Before analysis, data quality checks were performed on all 5 tables:

- Identified and removed **NULL values** from critical columns across all tables
- Disabled safe update mode (`SET SQL_SAFE_UPDATES = 0`) for cleaning operations
- Removed incomplete delivery records with missing `delivery_status`, `delivery_time`, or `rider_id`

## 📊 Business Analysis — 18 Questions

### 👥 Customer Analysis

| # | Question | Concepts Used |
|---|---|---|
| 1 | Top 5 most frequently ordered dishes by a specific customer in the last 1 year | `JOIN`, `WHERE`, `GROUP BY`, `ORDER BY`, `LIMIT` |
| 3 | Average order value per customer who placed more than 10 orders | `JOIN`, `GROUP BY`, `HAVING` |
| 4 | Customers who spent more than ₹5000 total on food | `JOIN`, `GROUP BY`, `HAVING`, `SUM` |
| 12 | Customer segmentation — Gold / Neutral / Cheaper based on avg order value | `CASE WHEN`, Correlated Subquery, `AVG` |
| 15 | Total revenue generated by each customer | `GROUP BY`, `SUM` |

---

### 🏪 Restaurant Analysis

| # | Question | Concepts Used |
|---|---|---|
| 5 | Orders placed but not delivered (cancelled) per restaurant | `JOIN`, `GROUP BY`, `HAVING` |
| 6 | Rank restaurants by total revenue within each city | `RANK()`, Window Function, `PARTITION BY` |
| 7 | Most popular dish in each city by order count | `JOIN`, `GROUP BY`, `COUNT` |
| 9 | Order cancellation rate comparison — current year vs previous year | `CASE WHEN`, `NULLIF`, `ROUND`, `YEAR(CURDATE())` |
| 11 | Restaurant growth ratio based on total delivered orders | `JOIN`, `GROUP BY`, `COUNT` |
| 14 | Peak order day of the week for each restaurant | `RANK()`, `DATE_FORMAT`, Window Function, Subquery |

---

### 🛵 Rider Analysis
